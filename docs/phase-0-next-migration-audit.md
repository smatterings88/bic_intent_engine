# Phase 0 ‚Äî Next.js Migration Audit

## Current Repo Summary

This repository is a **TanStack Start** application (TanStack Router file-based routes + `@tanstack/react-start`) built with **Vite 7**, not a minimal client-only Vite SPA. Configuration is delegated to **`@lovable.dev/vite-tanstack-config`**, which bundles TanStack Start, React, Tailwind CSS v4, Cloudflare build integration, path aliases, and related plugins. The production build emits both **`dist/client`** and **`dist/server`** (SSR/worker-oriented output).

**Stack (high level):** React 19, TypeScript (strict), TanStack Router, TanStack Start, Tailwind CSS **4.2** via `@tailwindcss/vite`, **shadcn/ui** (New York style) on **Radix UI**, `class-variance-authority`, `tailwind-merge`, `lucide-react`, and many optional UI dependencies. **Cloudflare** is represented by `@cloudflare/vite-plugin` in dependencies and **`wrangler.jsonc`** pointing at `@tanstack/react-start/server-entry`.

**Source layout:** `src/routes/` defines public pages; `src/components/SiteLayout.tsx` wraps all routes with header/footer; `src/styles.css` holds Tailwind v4 theme tokens and global styles; `src/components/ui/` contains generated shadcn-style primitives; `src/lib/utils.ts` exports `cn()`; `src/hooks/use-mobile.tsx` is a client-only breakpoint hook. **`src/routeTree.gen.ts`** is auto-generated by TanStack Router and must not be hand-edited.

**Public static assets:** `public/favicon.png` only (referenced from root head as `/favicon.png`).

## Migration Risks

| Risk                                   | Where Found                                                                   | Recommended Fix                                                                                                                                                                   |
| -------------------------------------- | ----------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **TanStack Router coupling**           | All `src/routes/*.tsx`, `SiteLayout.tsx`, `router.tsx`, `routeTree.gen.ts`    | Replace with **Next App Router** file routes; swap `Link`/`createFileRoute`/`head` for Next `Link`, `page.tsx`, `layout.tsx`, **`metadata`**.                                     |
| **`import.meta.env`**                  | `src/router.tsx`                                                              | Use `process.env.NODE_ENV` or Next `import.meta.env` only if configured; prefer Next conventions for dev-only error details.                                                      |
| **Lovable / Cloudflare-centric build** | `vite.config.ts`, `package.json`, `wrangler.jsonc`, dual client/server `dist` | Vercel target will use **Next build**, not Wrangler worker entry; retire or replace Cloudflare-specific pipeline for production.                                                  |
| **External OG/Twitter images**         | `src/routes/__root.tsx`                                                       | Host stable preview images on your domain; wire **`metadata.openGraph.images`** / **`twitter.images`**.                                                                           |
| **Google Fonts via `<link>`**          | `__root.tsx`                                                                  | Prefer **`next/font`** (Google) for self-contained loading and layout stability.                                                                                                  |
| **`window` / `document` in shared UI** | `src/hooks/use-mobile.tsx`; `src/components/ui/sidebar.tsx`                   | Mark **`"use client"`**; ensure no import from Server Components unless through client boundary. Public site does not currently import `sidebar` from pages‚Äîlow immediate risk.   |
| **Contact form is non-functional**     | `src/routes/contact.tsx`                                                      | `preventDefault` + local state only; Phase 1+ should use **Server Actions** or **API route** + validation (out of Phase 0 scope).                                                 |
| **Large static content in TSX**        | Multiple routes                                                               | For ISR/articles later, move long copy to **MDX/CMS/Zenith payloads**; for Phase 1 port, copying TSX is acceptable to preserve fidelity.                                          |
| **Dynamic `[slug]` vs reserved paths** | Future `/[landingPageSlug]`                                                   | Next.js **catch-all order** and reserved segments (`/articles`, `/app`, etc.) need a routing plan; consider **`(marketing)` route group** or middleware to avoid slug collisions. |
| **`@tanstack/react-query` unused**     | `package.json`                                                                | Either use in app layer later or omit from Next package.json to reduce bundle noise.                                                                                              |
| **Dependency audit noise**             | `npm install` reported vulnerabilities                                        | Run `npm audit` in Phase 1; not blocking current build.                                                                                                                           |
